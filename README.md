# Cpp-OOP-Shape-Polymorphism
# C++ Object-Oriented Programming (OOP): Geometry & Polymorphism

## 📌 About the Project
This repository contains a classic C++ Object-Oriented Programming (OOP) architecture demonstrating inheritance, encapsulation, and runtime polymorphism. By modeling a hierarchy of geometric shapes, this project showcases the ability to design scalable, modular, and maintainable C++ code—a mandatory skill for modern software engineering and embedded systems.

## ⚙️ Core Features
* **Class Inheritance:** Implements a base class (`Figura`) with derived concrete classes (`Cerc`, `Triunghi`, `Dreptunghi`).
* **Polymorphism:** Utilizes virtual functions to dynamically compute properties based on the specific object type at runtime.
* **Header & Source Separation:** Demonstrates professional C++ project structuring by separating class declarations (`.h` files) from their implementations (`.cpp` files).

## 🛠️ Tech Stack & Concepts
* **Language:** C++
* **Concepts:** Object-Oriented Programming (OOP), Inheritance, Polymorphism, Virtual Methods, Modular Architecture.

## 📁 Repository Structure
* `Figura.h` / `Figura.cpp` - The base class defining the standard interface for all shapes.
* `Cerc.h` / `Cerc.cpp` - Derived class for Circle implementations.
* `Triunghi.h` / `Triunghi.cpp` - Derived class for Triangle implementations.
* `Dreptunghi.h` / `Dreptunghi.cpp` - Derived class for Rectangle implementations.
* `main.cpp` - The main executable demonstrating object instantiation and polymorphic behavior.
